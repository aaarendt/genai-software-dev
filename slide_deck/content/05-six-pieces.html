Every AI coding assistant has all of these six components.

LLM backbone

The language model itself — the reasoning engine that turns prompts into text, code, and plans. Every AI tool you use is built on top of one.

Tool use

The ability to read files, edit them, and run shell commands on your machine. Without tool use the model can only talk; with it the model can act.

Agent loop

The software that decides whether to call another tool or return a result to you. It turns a single prompt into a sequence of goal-directed actions.

Project memory

A file the agent reads on startup so it knows your codebase, conventions, and working style. You write it once; every session starts with that context already loaded.

MCP servers

A standard protocol for connecting external systems — databases, calendars, APIs — directly to an agent. Any service that speaks MCP becomes immediately available to any tool that supports the standard.

Skills

Reusable procedures the agent can invoke for a specific kind of task. Instead of re-explaining how to deploy or review code each time, you capture that workflow once.
