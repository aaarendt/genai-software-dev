Verification

There are multiple approaches.
Today we used two.

  • Read the diff, not the chat. The chat tells you what the agent claims it changed. The diff is where you confirm it.
  • Push back. The agent is a coworker, not an oracle. "No, you're in the wrong mode" and "no, you did more than I asked" are both valid pushback. Half of what looks like the agent going rogue is a permissions or scope problem.

There are more advanced patterns — evals, test-first workflows, structured uncertainty flagging. The handout has starting points if you want to go deeper.
